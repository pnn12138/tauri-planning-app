# 修改任务卡片样式以符合设计规范

## 目标
根据提供的设计文件，修改当前任务卡片的样式，移除内部按钮，使其与设计规范一致，同时确保响应式设计、视觉细节优化和性能考虑。

## 分析
1. **设计规范**：任务卡片应包含标签、标题和底部元数据，没有内部操作按钮
2. **当前实现**：卡片内部包含开始/停止按钮，样式与设计不符
3. **主要文件**：
   - `src/Home.tsx`：包含任务卡片渲染逻辑
   - `src/App.css`：包含任务卡片样式

## 实施步骤

### 1. 修改任务卡片渲染逻辑 (`src/Home.tsx`)
- 更新 `renderTaskCard` 函数，移除内部的开始/停止按钮
- 调整卡片结构，使其分为主要内容区和底部元数据区
- 简化卡片内部元素，只保留标签、标题和元数据
- 确保拖拽功能正常工作
- 确保点击卡片查看详情功能正常

### 2. 更新任务卡片样式 (`src/App.css`)
- 添加 `task-card` 类的基础样式，包括：
  - 阴影和悬停效果
  - 圆角设计
  - 边框样式
  - 过渡动画
- 实现响应式设计，确保卡片在不同设备上正常显示
- 调整卡片布局为垂直结构，使用 Flexbox 实现
- 更新标签样式，使其与设计一致（颜色、字体大小、圆角）
- 添加底部元数据区样式，包括：
  - 背景色区分
  - 元数据项布局
  - 图标样式
- 为不同状态的卡片设计差异化样式：
  - 进行中：蓝色边框、特殊阴影效果
  - 已完成：灰色背景、删除线文本
  - 待做：标准样式

### 3. 性能优化考虑
- 确保卡片渲染高效，避免不必要的重渲染
- 对于大量任务场景，考虑使用虚拟滚动技术（如 React Virtualized）

### 4. 测试与回归验证
- 测试拖拽功能是否正常
- 测试点击卡片查看详情功能是否正常
- 测试状态切换功能是否正常
- 测试在不同设备尺寸下的显示效果
- 与设计师沟通确认视觉效果符合预期

## 预期效果
- 任务卡片样式与设计文件一致
- 移除了内部操作按钮
- 保留了所有现有功能
- 提高了卡片的视觉一致性和美观度
- 实现了响应式设计，适配不同设备
- 优化了视觉细节，提升了用户体验
- 确保了性能和功能的稳定性
