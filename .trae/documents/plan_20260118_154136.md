## 问题分析

1. **根本原因**：`buildTimelineModel` 函数总是使用当前日期过滤任务，导致无法显示其他日期的任务
2. **具体表现**：
   - 函数第162行使用固定的 `new Date()` 创建 `today` 变量
   - 第171-176行只保留与当前日期相同的任务
   - 无论传入什么日期的 `timelineTasks`，都只显示当前日期的任务
3. **相关代码位置**：
   - `buildTimelineModel` 函数：`src/shared/timeline/timelineDomain.ts:160`
   - 任务过滤逻辑：`src/shared/timeline/timelineDomain.ts:171-176`

## 解决方案

### 1. 修改 `buildTimelineModel` 函数，添加 `baseDate` 参数

修改 `buildTimelineModel` 函数，添加一个 `baseDate` 参数，允许传入指定的日期：

```javascript
export const buildTimelineModel = (tasks: Task[], config: TimelineConfig, baseDate: Date = new Date()): TimelineModel => {
  const { dayStart, dayEnd, minSlotMinutes } = config;
  // 使用传入的 baseDate 而不是固定的 new Date()
  const dayStartDate = createDateForTime(dayStart, baseDate);
  const dayEndDate = createDateForTime(dayEnd, baseDate);
  // ... 其他逻辑不变
};
```

### 2. 在 `Home.tsx` 中调用 `buildTimelineModel` 时传入 `selectedDate`

修改 `Home.tsx` 中的调用，将 `selectedDate` 作为 `baseDate` 传入：

```javascript
const timelineModel = useMemo(() => {
  return buildTimelineModel(timelineTasks, timelineConfig, new Date(selectedDate));
}, [timelineTasks, timelineConfig, selectedDate]);
```

### 3. 确保 `timelineTasks` 包含所选日期的所有任务

验证 `loadTodayData` 函数是否真的返回了所选日期的任务，而不仅仅是今天的任务。
