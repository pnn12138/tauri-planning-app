## 问题分析
从terminal输出可以看到，每当创建任务后，客户端会自动重启，并提示 `Info File src-tauri\data\planning.db changed. Rebuilding application...`。

**根本原因**：
- `src-tauri/src/paths.rs` 中的 `get_app_config_dir` 函数硬编码返回 `./data` 目录
- 这导致数据库文件 `planning.db` 被存储在 `src-tauri/data/` 目录下
- Tauri开发服务器监听 `src-tauri` 目录变化，数据库修改触发了自动重启

## 解决方案
修改 `get_app_config_dir` 函数，使用Tauri提供的 `app.path().app_data_dir()` 方法获取系统应用数据目录，将数据库文件存储在项目目录外，避免触发Tauri的文件监听器。

## 修复步骤
1. 修改 `src-tauri/src/paths.rs` 文件中的 `get_app_config_dir` 函数
2. 替换硬编码的 `./data` 目录为 `app.path().app_data_dir()` 的返回值
3. 保持函数签名不变，确保现有调用代码无需修改
4. 验证修复效果，确保任务创建后应用不再自动重启

## 预期效果
- 数据库文件将存储在系统应用数据目录（如Windows的 `%APPDATA%/tauri-planning-app/`）
- 任务创建时不会触发Tauri应用自动重启
- 所有现有功能保持不变，仅改变数据库存储位置

## 代码修改点
```rust
// src-tauri/src/paths.rs
pub fn get_app_config_dir(app_handle: &AppHandle) -> Result<PathBuf, crate::ipc::ApiError> {
    // 使用Tauri提供的API获取应用数据目录
    let config_dir = app_handle.path().app_data_dir().ok_or_else(|| crate::ipc::ApiError {
        code: "ConfigDirNotFound".to_string(),
        message: "Failed to get application data directory".to_string(),
        details: None,
    })?;
    
    // 确保目录存在
    std::fs::create_dir_all(&config_dir).map_err(|e| crate::ipc::ApiError {
        code: "ConfigDirNotFound".to_string(),
        message: format!("Failed to create config directory: {}", e),
        details: None,
    })?;
    
    Ok(config_dir)
}
```
