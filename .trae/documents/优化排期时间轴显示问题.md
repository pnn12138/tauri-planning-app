## 问题分析
当前排期版的主要问题是：大部分可安排的空闲块会被覆盖底部的时间，导致用户无法看到完整的时间范围。

## 原因分析
1. 时间轴容器高度固定，空闲块高度是根据持续时间计算的百分比
2. 当一天中大部分时间都是空闲时，空闲块高度会很大，覆盖底部时间刻度
3. 时间块（尤其是空闲块）直接渲染在容器中，没有考虑容器高度限制

## 解决方案
参考 `stitch_integrated_timeline_kanban_view (9)` 的 UI 界面，实现优化：

### 1. 调整时间轴渲染层次
- 将时间刻度层（ticks layer）置于最高层级，确保始终可见
- 调整时间块层（blocks layer）的渲染方式，避免覆盖时间刻度

### 2. 优化空闲块显示逻辑
- 修改空闲块的高度计算，确保不超出容器范围
- 实现空闲块的滚动机制，允许用户查看完整时间范围
- 添加最小高度限制，确保每个空闲块都能清晰显示

### 3. 改进时间轴容器布局
- 调整时间轴容器的 CSS 样式，确保内容正确显示
- 添加适当的溢出处理，允许垂直滚动查看完整时间轴

### 4. 优化拖拽交互体验
- 确保拖拽指示器始终可见
- 优化拖拽过程中的视觉反馈

## 需要修改的文件
1. `src/Home.tsx` - 调整时间轴渲染逻辑和样式
2. `src/shared/timeline/timelineDomain.ts` - 优化空闲块计算逻辑
3. 相关 CSS 文件 - 调整时间轴样式

## 具体实现步骤
1. 调整时间轴渲染层次，将时间刻度层置于顶层
2. 修改空闲块的高度计算方式，添加最大高度限制
3. 实现时间轴容器的垂直滚动功能
4. 优化拖拽指示器的显示位置
5. 测试并调整样式，确保符合参考设计

## 预期效果
- 时间刻度始终可见，不会被空闲块覆盖
- 空闲块高度适中，清晰显示可安排时间
- 时间轴支持垂直滚动，可查看完整时间范围
- 拖拽交互流畅，视觉反馈清晰

这个解决方案将解决当前排期版的显示问题，提供更好的用户体验，符合参考设计的要求。
